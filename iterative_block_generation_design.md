# FlashMTP v3.1 迭代式块内生成设计

## 1. 目标与范围

本文档给出一套新的 `FlashMTP v3.1` 设计说明，目标是在不为草稿模型保留 KV cache 的前提下，利用大模型最新融合 hidden states 作为条件，在一个长度为 `B` 的 draft block 内进行最多 `N` 轮迭代生成。

本文档只讨论以下问题：

- 草稿模型如何在块内分轮生成，而不是一次前向直接承诺全部 `B` 个 token。
- 训练目标如何改写为基于 `clean prefix` 的 completion 学习。
- 如何设计前重后轻、且对不同剩余长度归一化的损失权重。
- 如何定义两类块内截断准则：启发式方案与可学习方案。
- 如何与现有 `FlashMTP` 推理验收机制对齐。

本文档不采用当前 `v3` 里的蒸馏损失与一致性损失叙事，也不要求块内预测时让大模型参与中间验证。大模型仍只在最终 speculative decoding 验收阶段发挥作用。

## 2. 核心假设

本文档沿用以下基本假设，不再动摇：

1. 大模型最新融合 hidden states 已经编码了历史上下文的关键信息，草稿模型可以仅依赖这些条件而不保留自己的 KV cache。
2. 不同层的 hidden states 关注历史的不同方面，因此多层条件信息的融合仍然是有价值的。
3. 一次性预测整个长度为 `B` 的 block，后半段更容易因为语义转折和条件不足而出错；因此更合理的方式是先预测一段可靠前缀，再基于该前缀继续预测剩余部分。
4. 在 speculative decoding 中，最终收益取决于大模型能连续接受多长的前缀，而不是草稿模型在整块上的平均 token 精度。

## 3. 设计动机

当前一次前向直接预测整个 `B` 长块的方式有一个根本困难：草稿模型必须在没有中间自条件更新的情况下，同时决定块内所有位置的 token。对于越靠后的位置，这种决策既依赖条件 `CHS`，又依赖块内前部 token 的具体取值，而后者在单次前向中并未被真正“确认”。

因此，更合理的目标不是：

- 一次前向把整个块全部预测正确。

而是：

- 给定条件 `CHS` 和一个已确认的块内前缀，可靠地补全剩余后缀；
- 在推理时，通过若干轮块内迭代，把“全未知”逐步转化为“已确认前缀 + 未知后缀”。

这使得草稿模型的任务从“一步猜全段”变成“多步扩展连续前缀”。这与当前 speculative decoding 的验收机制天然一致，因为当前 target 侧本来就是按最长连续前缀来接受草稿 token。

## 4. 形式化定义

### 4.1 条件与目标块

记：

- `c` 表示由大模型最新融合 hidden states 得到的条件表示。
- 目标 draft block 为
  $
  x_{1:B} = (x_1, x_2, \ldots, x_B).
  $
- `B` 是单个 draft block 的长度。
- `N` 是块内迭代的最大轮数。
- `[M]` 表示 mask token。

### 4.2 第 `t` 轮块内状态

记第 `t` 轮已经确认的前缀长度为 `p_t`，满足
$
0 \le p_t \le B.
$

则第 `t` 轮输入状态为
$
z^{(t)} = (x_1, \ldots, x_{p_t}, [M], \ldots, [M]).
$

也就是说：

- 前 `p_t` 个位置使用 clean token；
- 后 `B - p_t` 个位置全部为 mask。

草稿模型在条件 `c` 与状态 `z^{(t)}` 下输出块内各位置的分布：
$
q_\theta^{(t)}(\cdot)
=q_\theta(\cdot \mid c, z^{(t)}).
$

对于任意尚未确定的位置 `i > p_t`，模型给出：
$
q_\theta(x_i \mid c, x_{1:p_t}, [M]_{p_t+1:B}).
$

### 4.3 块内迭代更新

在第 `t` 轮，模型先对剩余区间 `p_t+1 : B` 产生预测，再由某个截断准则决定本轮接受多少个新增 token。记该新增长度为 `k_t`，满足
$
0 \le k_t \le B - p_t.
$

更新规则为
$
p_{t+1} = p_t + k_t.
$

若 `p_t = B`，则说明整个 block 已完成；若达到最大轮数 `N` 仍未完成，则由工程策略决定是停止、退化为更保守的接受方式，还是退回最小步长继续生成。

## 5. 训练目标：基于 clean prefix 的 completion 学习

### 5.1 为什么训练前缀必须使用 clean token

这里不引入 prefix 误差鲁棒性训练。理由很直接：

- 在推理时，只有被大模型连续接受的前缀才会成为下一轮的条件；
- 一旦前缀某处出错，target 侧就不会继续接受后续 token；
- 因此，下一轮块内生成真正会条件化到的前缀，应该被视为“已经确认正确的前缀”。

所以训练目标应与实际使用场景严格对齐：前缀部分始终使用 clean token，而不是模型自生成且可能出错的 prefix。

### 5.2 训练样本构造

给定一个目标块 `x_{1:B}`，随机采样一个切分点
$
p \sim \mathcal{P}, \qquad 0 \le p \le B-1.
$

构造训练输入状态：
$
z^{(p)} = (x_1, \ldots, x_p, [M], \ldots, [M]).
$

记剩余长度为
$
R = B - p.
$

模型在条件 `c` 与状态 `z^{(p)}` 下，对剩余位置给出预测：
$q_\theta(\cdot \mid c, z^{(p)})_{p+d}, \qquad d = 1, 2, \ldots, R.$

### 5.3 主损失

定义块内 completion 主损失为
$\mathcal{L}_{\mathrm{pred}}(c, x_{1:B}; p)=
\sum_{d=1}^{R}
\tilde w_d^{(R)}
\cdot
\mathrm{CE}
\Big(
q_\theta(\cdot \mid c, z^{(p)})_{p+d},
x_{p+d}
\Big).
$

整体训练目标为
$\mathcal{L}_{\mathrm{train}}
=\mathbb{E}_{(c, x_{1:B}) \sim \mathcal{D}}
\,\mathbb{E}_{p \sim \mathcal{P}}
\Big[
\mathcal{L}_{\mathrm{pred}}(c, x_{1:B}; p)
\Big].$

其中：

- `CE` 是标准 token-level 交叉熵；
- 每个剩余位置都参与监督；
- 权重 $`\tilde w_d^{(R)}`$ 用于强调“越靠前的未决位置越重要”。

## 6. 归一化的平滑距离衰减权重

### 6.1 指数衰减形式


最推荐的默认形式是指数衰减：
$w_d = \exp\!\left(-\frac{d-1}{\gamma}\right), \qquad d = 1, 2, \ldots, R,$
并做剩余长度归一化：
$\tilde w_d^{(R)}=\frac{w_d}{\sum_{j=1}^{R} w_j}.$

这里 $`\gamma > 0`$ 是衰减温度：

- $`\gamma`$ 越小，前几位的权重越集中；
- $`\gamma`$ 越大，权重越平缓。

这个形式与 `FlashMTP_v1.2` 中已有的位置衰减思路一致，只不过原实现是按块内绝对位置定义，而这里改写为按“剩余后缀内距离”定义。对于任意切分点 `p`，剩余后缀中的第一个未决位置 `d = 1` 始终拥有最大权重 `1.0`。

### 6.2 幂律衰减形式

另一种可选写法是幂律衰减：
$w_d = d^{-\alpha}, \qquad \alpha > 0,$

$\tilde w_d^{(R)}=\frac{w_d}{\sum_{j=1}^{R} w_j}.
$

幂律衰减比指数衰减更“长尾”，会给后部位置保留更多监督权重。如果实验上发现指数衰减让模型过度关注最前面的少数位置，可以尝试切换到幂律形式。

### 6.3 为什么需要归一化

若不对 `w_d` 做归一化，不同切分点 `p` 对应的剩余长度 `R` 会直接改变总损失尺度，从而引入额外偏置：

- `R` 较大的样本会天然贡献更大的总 loss；
- `R` 较小的样本会被低估。

归一化后：
$
\sum_{d=1}^{R} \tilde w_d^{(R)} = 1,
$
于是不同剩余长度下的训练样本在总量级上可比，优化器看到的是“同一类 completion 任务的不同难度实例”，而不是“长度越长权重越大”的混合目标。

### 6.4 数学性质

对固定的切分点 `p` 和正权重 $`\tilde w_d^{(R)} > 0`$，上式仍然是一个加权的 proper scoring objective。也就是说，在理想函数类和总体期望意义下，最优解仍然要求每个位置的预测分布匹配真实条件分布；权重改变的是有限模型容量和有限数据下的优化侧重点，而不是 token-level 条件分布的 Bayes 最优形式。

因此，前重后轻权重的作用不是改变正确答案，而是显式告诉模型：

- 如果容量有限，优先把“更早的未决位置”做准；
- 因为这些位置更直接决定最终可接受前缀长度。

## 7. 块内迭代推理算法

### 7.1 推理目标

推理时不再要求一次前向直接生成整个 `B` 长块，而是通过最多 `N` 轮迭代，逐步扩展块内前缀。

设第 `t` 轮已经接受的前缀长度为 `p_t`，则：

1. 输入状态为 $`x_{1:p_t}`$ 加上右侧全 mask；
2. 模型预测剩余区间 `p_t+1 : B`；
3. 截断准则给出本轮新增长度 `k_t`；
4. 将前 `k_t` 个新增 token 视作下一轮的 clean prefix；
5. 直到块完成或达到最大轮数 `N`。

### 7.2 数学描述

记第 `t` 轮模型对剩余位置的点预测为
$
\hat x_{p_t+1:B}^{(t)}.
$

截断准则给出
$
k_t = \mathrm{Cutoff}\!\left(
q_\theta^{(t)},
\hat x_{p_t+1:B}^{(t)},
p_t
\right).
$

然后定义下一轮状态为
$z^{(t+1)}=
\Big(
\hat x_1, \ldots, \hat x_{p_t+k_t},
[M], \ldots, [M]
\Big),
$
其中前缀部分由前几轮累计接受的 token 构成。

### 7.3 伪代码

```text
Input: 条件 c, block size B, max rounds N
Initialize: p = 0, block = [M, ..., M]

for t in 0 .. N-1:
    logits = DraftModel(c, block_with_clean_prefix(block, p))
    pred = Decode(logits[p+1 : B])
    k = Cutoff(logits, pred, p)

    if k <= 0:
        执行保守回退策略
        break 或令 k = 1

    block[p+1 : p+k] = pred[1 : k]
    p = p + k

    if p == B:
        break

return block
```

### 7.4 工程上的保守回退

若某轮截断准则给出 `k_t = 0`，说明模型认为当前剩余区间整体都不够可靠。此时可以考虑三种保守策略：

1. 直接停止该 block 的草稿扩展，让 target 尽快接管；
2. 强制退化为 `k_t = 1`，让块内生成最小步长推进；
3. 降低阈值后重试一轮。

其中最稳健的通常是第 1 或第 2 种。第 3 种需要谨慎，否则容易把本来不可靠的预测硬塞进前缀。

## 8. 为什么“连续前缀”是唯一合理的块内接受单位

在当前 speculative decoding 验收机制中，大模型接受的是最长连续匹配前缀，而不是若干离散的高置信位置。因此块内迭代时也应只扩展连续前缀。

原因有三点：

1. target 侧的验收本来就是前缀式的。一旦前面某处错了，后面位置即使偶然正确，也不会被继续接受。
2. 若在块内做“散点 unmask”，则后续预测会条件化到一些尚未被 target 连续确认的位置，这与真实可执行推理路径不一致。
3. 连续前缀是唯一和现有验收规则完全同构的块内状态表示。

因此，块内状态不应被定义为“任意集合位置已知”，而应定义为“前 `p` 个位置已知，后面全 mask”。

## 9. 两类截断准则

截断准则的本质是：在没有块内 target 验证的情况下，估计本轮最多能安全扩展多长的连续前缀。

严格地说，理想对象是事件
$A_k^{(t)}=\mathbf{1}
\left[\hat x_{p_t+1:p_t+k}^{(t)}=x_{p_t+1:p_t+k}\right],$
即“本轮新增的前 `k` 个 token 是否全部正确”。

我们真正想估计的是
$
P\!\left(A_k^{(t)} = 1 \mid c, z^{(t)}\right).
$

下面给出两类方案。

### 9.1 方案 A：启发式截断

#### 9.1.1 基本思想

不额外训练新的 head，而是直接利用草稿模型 logits 的统计量构造每个位置的可靠性分数，再据此选出最长连续前缀。

对第 `t` 轮剩余区间中的第 `j` 个新增位置，可定义以下量：

- 最大概率
  $
  p^{\max}_{t,j} = \max_v q_\theta^{(t)}(v)_{p_t+j};
  $
- top-1 与 top-2 的 logit 间隔
  $
  m_{t,j} = \ell^{(1)}_{t,j} - \ell^{(2)}_{t,j};
  $
- 预测熵
  $
  H_{t,j}=\sum_v q_\theta^{(t)}(v)_{p_t+j}
  \log q_\theta^{(t)}(v)_{p_t+j}.
  $

基于这些量，可以构造启发式可靠性分数：
$
r_{t,j}=\beta_1 \log p^{\max}_{t,j}+ \beta_2 m_{t,j}- \beta_3 H_{t,j}- \beta_4 \frac{j-1}{B-p_t},$
其中 `\beta_1, \beta_2, \beta_3, \beta_4 \ge 0` 是可调系数。

若不想引入组合系数，也可以直接使用最简单的 `p^{\max}_{t,j}` 或 `-H_{t,j}` 作为可靠性分数。

#### 9.1.2 最大阈值前缀规则

给定阈值 `\tau_j`，取最大的 `k_t` 满足：
$r_{t,j} \ge \tau_j,\qquad \forall j = 1, 2, \ldots, k_t.
$

也就是：
$
k_t=
\max
\left\{
k :
r_{t,1}, \ldots, r_{t,k}
\text{ 全部通过阈值}
\right\}.
$

这里的阈值可以是：

- 常数阈值 $`\tau_j = \tau`$；
- 位置相关阈值 $`\tau_j = \tau(j)`$；
- 或者用一个常数阈值，再在 $`r_{t,j}`$ 中显式加入位置惩罚项。

#### 9.1.3 风险预算规则

也可以先把每个位置映射成启发式错误风险：
$
e_{t,j} = g(r_{t,j}),
$
其中 `g` 是一个单调递减映射，例如：
$
e_{t,j} = 1 - p^{\max}_{t,j}.
$

然后定义累计风险预算$ `\epsilon`$，取最大的$ `k_t`$ 满足
$
\sum_{j=1}^{k_t} e_{t,j} \le \epsilon.
$

这个规则比逐点阈值更灵活，因为它允许前若干位置中存在轻微波动，只要累计风险仍在预算内。

#### 9.1.4 优缺点

优点：

- 不需要额外训练新的参数；
- 易于实现，适合作为第一版 sanity check；
- 能快速验证“块内迭代生成”本身是否带来收益。

缺点：

- raw confidence 往往未校准；
- 阈值依赖模型规模、温度、采样方式和位置分布；
- 位置之间的相关性没有被显式建模，不能直接等价为 prefix 正确概率。

### 9.2 方案 B：可学习截断

#### 9.2.1 基本思想

引入一个轻量的 acceptance head 或 boundary head，让模型显式预测“本轮最多有多长的新增前缀会整体正确”。

有两种建模方式：

1. token-level 正确率预测；
2. prefix-level 存活概率预测。

其中更推荐第二种，因为它与 speculative decoding 的连续前缀验收机制直接同构。

#### 9.2.2 token-level 方案

对第 `t` 轮剩余区间中第 `j` 个位置，预测：
$
s_{t,j}=
P\!\left(
\hat x_{p_t+j}^{(t)} = x_{p_t+j}
\mid c, z^{(t)}
\right).
$

监督标签可以直接由草稿预测与真值比较得到：
$
y_{t,j}=
\mathbf{1}
\left[
\hat x_{p_t+j}^{(t)} = x_{p_t+j}
\right].
$

训练损失为二元交叉熵：
$
\mathcal{L}_{\mathrm{tok}}=- \frac{1}{R_t}
\sum_{j=1}^{R_t}
\Big(y_{t,j}\log s_{t,j}+ (1-y_{t,j})\log (1-s_{t,j})
\Big),
$
其中 $`R_t = B - p_t`$。

然后可定义一个近似前缀评分，例如
$
\tilde S_{t,k}=
\prod_{j=1}^{k} s_{t,j},
$
再按阈值决定 `k_t`。

这种方法实现简单，但它默认位置间独立，通常不是最理想的前缀建模方式。

#### 9.2.3 prefix-level 方案

更推荐直接建模 prefix 存活概率：
$
S_{t,k}=
P\!\left(
\hat x_{p_t+1:p_t+k}^{(t)}=
x_{p_t+1:p_t+k}
\mid c, z^{(t)}
\right),
\qquad
k = 1, \ldots, R_t.
$

监督标签定义为
$
Y_{t,k}=\mathbf{1}\left[\hat x_{p_t+1:p_t+k}^{(t)}
=x_{p_t+1:p_t+k}
\right].
$

对应损失可写为
$
\mathcal{L}_{\mathrm{prefix}}=- \frac{1}{R_t}
\sum_{k=1}^{R_t}\Big(Y_{t,k}\log S_{t,k}+ (1-Y_{t,k})\log (1-S_{t,k})
\Big).
$

推理时取最大的 `k_t` 满足
$
S_{t,k_t} \ge \tau.
$

#### 9.2.4 单调性约束

prefix 存活概率应满足：
$
S_{t,1} \ge S_{t,2} \ge \cdots \ge S_{t,R_t}.
$

因为“前 `k+1` 个全对”的事件必然蕴含“前 `k` 个全对”。

为保证这一性质，可以使用两种方式：

1. 直接输出 $`S_{t,k}`$，并加一个单调性正则项；
2. 输出条件存活因子 $`h_{t,j} \in (0,1)`$，再定义
   $
   S_{t,k} = \prod_{j=1}^{k} h_{t,j}.
   $

第二种方式天然保证单调性，更适合作为工程实现。

#### 9.2.5 为什么可学习方案不需要块内 target 验证

注意这里的监督目标不是“target 在块内会接受到哪里”，而是“草稿模型本轮新增前缀实际是否正确”。因此标签可以直接通过与 ground-truth block 对比得到，不需要在训练时让大模型逐步参与块内验证。

也就是说：

- 可学习截断 head 学的是“正确性概率估计”；
- target 侧仍然只在最终 speculative decoding 验收时出现。

#### 9.2.6 优缺点

优点：

- 直接针对“连续前缀能否整体正确”建模；
- 更容易和最终验收收益对齐；
- 比启发式阈值更有希望获得稳定的截断边界。

缺点：

- 需要额外参数和额外监督；
- 需要验证集做校准；
- 若 head 与主模型耦合不当，可能出现“主预测变强但边界估计滞后”的问题。

## 10. 数学原理与合理性说明

### 10.1 训练与推理状态的一致性

本设计最重要的结构约束是：训练和推理都使用“连续 clean prefix + 右侧全 mask”的状态表示。

这保证了以下一致性：

- 训练时见到的条件状态，与推理时真实会进入下一轮的条件状态同类；
- 不会让模型在训练时依赖一些推理时永远不会被保留的错误 prefix；
- 块内状态空间从一般的 mask 子集，收缩为一条简单的前缀链：
  $
  p = 0 \rightarrow 1 \rightarrow 2 \rightarrow \cdots \rightarrow B.
  $

这条链恰好对应 speculative decoding 的连续前缀接受逻辑。

### 10.2 为什么前重后轻 loss 是合理的

最终 speculative decoding 的收益本质上由“能多向前推进多少个连续正确 token”决定。块内第一个错误位置往往决定 target 本轮接受停止的边界。因此：

- 越靠前的未决位置，价值越高；
- 越靠后的未决位置，虽然仍然重要，但对单轮 acceptance boundary 的边际贡献更弱。

因此加权交叉熵不是随意拍脑袋的启发式，而是对最终系统收益的一种代理优化。

### 10.3 为什么不能把 token 概率直接当成 prefix 可接受概率

若仅看单个位置的 `top-1 probability`，它回答的是：

- “这个位置自己像不像对的？”

但真正决定是否应当继续接受的是：

- “前 `k` 个新增 token 是否会整体连续正确？”

这两者不是同一事件，因为前缀正确事件是一个联合事件。即使每个单点 token 概率都不低，其联合前缀正确概率也可能显著下降。因此，截断准则必须被单独处理，要么用启发式近似联合风险，要么直接用可学习的 survival head 去建模。

### 10.4 哪些结论是严格的，哪些只是原理性判断

可以严格陈述的部分：

- clean-prefix 训练与连续前缀状态空间完全一致；
- 归一化加权交叉熵在正权条件下仍保持每个位置的 proper scoring 性质；
- prefix-survival 概率应当关于 `k` 单调不增。

属于原理性判断而非一般性定理的部分：

- 块内越靠后的位置通常更难预测；
- 迭代式生成通常比一次性整块预测更适合处理语义转折；
- 某一种具体截断策略一定优于另一种。

这些结论应由实验验证，而不是当作无需检验的数学定理。

## 11. 与现有代码的对应关系

### 11.1 当前背景文档

- `progress.md` 记录的是总体研究背景和核心假设。
- `v3_modification_plan.md` 记录的是当前基于蒸馏 / 一致性的 `v3` 路线。

本文档对应的是另一条更贴近“迭代式块内生成”的路线，核心目标不再是 `y / y*` 上的 consistency，而是“任意 clean prefix 条件下的 completion 能力”。

### 11.2 训练包装入口

当前训练包装逻辑位于：

- `specforge/core/flashmtp.py`

这里已经具备：

- 采样 anchor；
- 构造 block 级输入；
- 基于 `mask token` 构造块内状态；
- 计算训练损失。

但当前 `v3.1` 主训练路径仍然围绕 `forward_v3()` 的蒸馏 / 一致性形式组织。若后续要实现本文档中的方案，主要需要把训练目标改写为本文件中的 clean-prefix completion 形式。

### 11.3 现有位置衰减参考

当前 `FlashMTP_v1.2/specforge/core/flashmtp.py` 中已经存在指数型块内衰减：
$
\exp\!\left(-\frac{\max(k-1, 0)}{\gamma}\right).
$

本文档建议保留同样的设计风格，但把坐标从“块内绝对位置”改写为“剩余后缀相对位置”。这样无论切分点 `p` 取何值，剩余后缀的第一个未决位置都始终获得最高权重。

### 11.4 现有推理验收规则

当前 draft 推理逻辑位于：

- `specforge/modeling/draft/flashmtp.py`

现有实现里，target 侧验收使用的是最长连续前缀匹配规则，本质上等价于：
$
\text{acceptance\_length}=\max\left\{k :\hat x_{1:k} = x^{\mathrm{target}}_{1:k}
\right\}.
$

实现上通过逐位相等后做 `cumprod` 来得到最长连续前缀。这意味着：

- 一旦前面某个 speculative token 不匹配，后面位置即使碰巧匹配也不会继续接受；
- 因此块内状态自然应该建模成连续前缀，而不是任意位置集合。

本文档提出的块内迭代生成只需要改变 draft block 的产生方式，不需要改变 target 侧“最长连续前缀接受”的验收原理。

## 12. 推荐实验顺序

为了降低实验复杂度，建议按以下顺序推进：

1. 先实现 clean-prefix 训练目标，不加额外截断 head。
2. 先用启发式截断做第一版块内迭代推理，验证该生成范式是否优于一次性整块预测。
3. 若块内迭代本身有效，再引入 prefix-survival head，替换启发式阈值。
4. 最后再系统比较：
   - 单轮整块生成；
   - 多轮块内迭代 + 启发式截断；
   - 多轮块内迭代 + 可学习截断。

## 13. 可实验变量

后续最值得系统扫描的变量包括：

- `B`：draft block 长度；
- `N`：块内最大迭代轮数；
- `\gamma` 或 `\alpha`：权重衰减超参数；
- 切分点 `p` 的采样分布；
- 每轮 token 解码方式：greedy / sampling；
- 启发式分数的组成；
- 截断阈值或风险预算；
- acceptance head 的结构与校准方式。

## 14. 小结

这条设计路线的核心不是让草稿模型“一步生成更长的 block”，而是让它学会：

- 在任意已确认前缀条件下，稳定补全剩余部分；
- 在推理时逐步扩展连续前缀；
- 把真正不确定的“截断边界”单独拿出来建模。

这样，草稿模型的生成任务、训练状态空间和 target 侧最终验收机制就三者对齐了。
