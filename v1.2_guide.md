# Plan：无 KV 草稿模型（单层一次性 Layer 交互 + 相对位置 Mask）

## 目标与约束

以下是对于现在的大模型hiddenstate沿着序列维度拼接的模式：seq mode的优化。**feature mode 分支直接删除**，简化代码，仅保留 seq mode。

- **条件**：仅使用主模型在**最后一 干净token 位置**的各层 hidden `H ∈ ℝ^{L×d_h}`（L为层数，无主模型 KV cache）。
- **草稿**：生成 `B` 个候选 token logits，**输出层与主模型词表共享**（沿用现有实现）。
- **Mask**：每个草稿槽位使用**特殊mask token**，**不再改这套嵌入本身**；将现有 **mask 绝对位置编码改为局部 RoPE 相对位置编码**（见 §Mask）。
- **非对称信息交互**：层hiddenstates tokens 与 mask tokens 可在**同一张注意力图**中计算，但用 **attention mask** 禁止层条件token→mask 的注意力（层不看 mask）；mask 可看层（cross 或统一注意力中的允许方向由 mask 定义）。

## 张量与序列构造

- **Layer tokens**（长度 `L`）：`z_l = RMSNorm(Proj(H_l)) + E_layer[l]`（`E_layer` 可训练 Layer-ID embedding）。
- **Mask（noise） tokens**（长度 `B`）：`m_b = E_mask[mask_token_id]`，**不加绝对位置**；在注意力层中通过 **RoPE 局部位置编码**（position_id = block 内 0..B-1）表达 mask 间的相对关系。
- **拼接序列**（不压缩长度）：`X = [z_0…z_{L-1}, m_0…m_{B-1}]`，总长 `L+B`。

## Layer token 表征：构造与作用

### 构造（每层一个 token，序列长度 = `L`）

对每个层索引 `l ∈ {0..L-1}`：

1. **线性对齐到草稿维度**：`h_l = W_proj · H_l`，其中 `W_proj` 是 `nn.Linear(H, H, bias=False)`。
2. **RMSNorm 归一化**：`h_l = RMSNorm(h_l)`，使用 `Qwen3RMSNorm`（与目标模型保持一致）。
3. **可训练 Layer-ID embedding**：`z_l = h_l + e_l`，其中 `e_l = E_layer[l] ∈ ℝ^{d_model}`，`E_layer ∈ ℝ^{L×d_model}`，是 `nn.Embedding(num_target_layers, hidden_size)`。

> **不要**把「全文绝对 token 位置」加在 layer tokens 上：位置语义应由主模型 hidden 内化；草稿侧用 `l` 只表示 **深度/层级身份**。

### CHS 提取布局（已确认：per-anchor 交错）

训练时从 target model 各层提取 anchor position - 1 处的 hidden states，**按 per-anchor 交错**排列：

```
[layer0_anchor0, layer1_anchor0, ..., layerL-1_anchor0, layer0_anchor1, layer1_anchor1, ..., layerL-1_anchor1, ...]
```

实现：`torch.stack(selected_states, dim=2).reshape(B, N*L, H)`（先 stack 出 layer 维度再 reshape，确保同一 anchor 的各层 token 相邻）。

> **注意**：v1.1 的 seq mode 使用 `torch.cat(selected_states, dim=1)` 是 per-layer 拼接，布局为 `[所有anchor的layer0 | 所有anchor的layer1 | ...]`，与 block mask 的 per-block 分组不一致，v1.2 已修复。

### 作用（为什么要 `E_layer`）

- **消歧**：`W_proj(H_l)` 在不同 `l` 上可能处于不同统计域/尺度；`E_layer[l]` 显式标记「这是第几层的条件向量」，降低注意力把不同层表征混成同一子空间的问题。
- **为轻量跨层交互服务**：后续 **仅一次**、**1 层 1 头** 的 layer self-attn 主要做跨层信息交换；`E_layer` 提供 **可分的深度索引**，让这次交互更稳定（否则多层向量更像「无标签的 L 个向量」）。
- **与 mask 分工**：layer 侧表达「多深度条件记忆」；mask 侧用固定 id + **局部 RoPE 位置**表达「草稿槽之间的局部关系」。

## Attention 非对称信息交互：用 attention mask 落地

定义注意力允许的边（以 Q 行、K 列为视角）：

- **Layer Q → Layer K**：允许（同一 block group 内全连接）。
- **Layer Q → Mask K**：**禁止**（实现为 FlexAttention 的 `mask_mod` 中 `~layer_to_mask`）。
- **Mask Q → Layer K**：允许（草稿槽从多层表征取信息）。
- **Mask Q → Mask K**：允许（同一 block 内双向）。

不同 block group 之间互不可见；无效 block（`block_keep_mask=False`）不参与注意力。

> 实现方式：统一大 mask（FlexAttention `create_block_mask`），在 `mask_mod` 函数中通过 `q_in_chs & ~kv_in_chs` 判断 Layer→Mask 边并禁止。

## Mask：局部 RoPE 相对位置（替换绝对位置）

### 已确认方案：RoPE 局部位置编码

**不使用**桶相对偏置表（Bias table），改为 **RoPE 仅作用于 mask 子序列**，position_id 为 block 内的局部位置 `[0, 1, ..., B-1]`：

- 每个 block 的 mask tokens 被分配 position_id = `0, 1, ..., B-1`（与全局绝对位置无关）。
- `rotary_emb` 从这些局部 position_id 生成 cos/sin。
- 在 attention 的 Q/K 投影后，**拆分** layer 部分和 mask 部分，**仅对 mask 部分**做 `apply_rotary_pos_emb`，然后再拼回。
- Layer tokens 的 Q/K **不做任何位置编码**。

```python
# Attention 中的 RoPE 拆分
q_layer, q_mask = q[:, :, :ctx_len, :], q[:, :, ctx_len:, :]
k_layer, k_mask = k[:, :, :ctx_len, :], k[:, :, ctx_len:, :]
q_mask, k_mask = apply_rotary_pos_emb(q_mask, k_mask, cos, sin)
q = torch.cat([q_layer, q_mask], dim=2)
k = torch.cat([k_layer, k_mask], dim=2)
```

### 与其它边的关系（保持非对称）

- **Layer→Layer**：允许，无位置编码（仅 Layer-ID embedding 区分深度）。
- **Layer→Mask**：禁止（`-inf` mask）。
- **Mask→Layer**：允许（草稿槽从多层条件取信息），无额外位置项。
- **Mask→Mask**：允许 + **局部 RoPE 只作用于这条边的 Q/K**。

## 推理适配（已确认）

- `**extract_context_feature`**：从 feature mode（`cat dim=-1`）改为 seq mode（`stack dim=2` + `reshape`），输出 `(B, S*L, H)` 按位置交错。
- **Draft model forward**：同训练路径，`fc` + `RMSNorm` + `layer_embedding` 应用于 target_hidden；`rotary_emb` 只对 noise_embedding 用局部 position_id。
- **移除 Draft KV cache**：新架构下每个 speculation step 的 layer tokens 都会更新（来自新的 target hidden states），draft model 的 KV cache 无法跨步复用。每步做独立的完整 forward，简化逻辑。
- **保留 Target KV cache**：target model 的 KV cache 机制不变。

## 实现注意

- RoPE **不需要** 作用在 layer tokens 上；layer 侧继续用 **Layer-ID embedding** 区分深度即可。
- `chs_concat_mode` 参数已彻底删除（训练脚本、启动脚本、模型代码），不再有 feature/seq 的选择。
- Layer token 的 `layer_ids` 生成方式：`torch.arange(L).repeat(N)`，即 `[0,1,...,L-1, 0,1,...,L-1, ...]` 重复 N 次（N 为 anchor 数量），与 per-anchor 交错布局对应。
