# Plan：无 KV 草稿模型（多层草稿 Transformer + 非对称注意力 + 局部 RoPE）

## 版本说明

- **v1.2.0**：为每层条件向量增加可训练 **Layer-ID embedding**（`E_layer`），经 `layer_emb_norm` 后在各草稿层注入。
- **v1.2.1**：将 Layer-ID 改为 **门控注入**：每层 decoder 在 `input_layernorm` 之后对 **layer 段**做 `c + σ(W_g c) ⊙ E_layer[l]`，mask 段不变，便于草稿网络从归一化隐状态中有选择地吸收深度信息。
- **v1.2.2**：**单次 RMSNorm**：`tilde_z = RMSNorm(W_proj·H + E_layer[l])`，去掉 `layer_emb_norm` 与 decoder **门控**（避免对 Layer-ID 重复归一化/重复注入）。
- **v1.2.3**：**进入 decoder 前不拼接** CHS 与 mask；每层 **`input_layernorm` 仅作用于 mask 段**，在 **计算 attention 之前**再 `concat([ctx, LN(mask)])`（后续 v1.2.4 改为仅 mask 为 Q、`ctx_init` 固定）。
- **v1.2.4**：**仅 mask 作为 attention 的 Q**；**`ctx_init`（初始 CHS）在各草稿层保持不变**，每层 `K/V = concat(ctx_init, LN(mask))`；残差与 **MLP 只更新 mask 流**；**稠密加性 mask** 由 **`build_flashmtp_dense_attn_bias`**（定义于 `specforge/core/flashmtp.py`，与 `create_flashmtp_block_mask` 同文件）在 **训练封装 / `spec_generate` 内提前构造** 后传入草稿 `forward`（不在 `FlashMTPDraftModel.forward` 内构造）。

## 目标与约束

本方案对应 **seq mode**：沿序列维展开多层主模型 hidden，**feature mode 已删除**，仅保留当前实现。

- **条件（CHS）**：从主模型 `hidden_states`（HF 顺序：索引 `0` 为 embedding 输出，索引 `k≥1` 为第 `k−1` 个 decoder 层之后）中，按 `target_layer_ids` 取 **若干层** 的向量，经投影与归一化后作为 **layer tokens**。主模型推理仍使用 **Target KV cache**；草稿侧 **不使用主模型 KV** 作为额外输入。
- **草稿**：对块内 mask 位置输出 logits，**与主模型共享 `lm_head`**。
- **Mask**：块内槽位使用 **mask token 嵌入**；**不做全文绝对位置编码**，对 mask 子序列使用 **局部 RoPE**（block 内 `0…B−1`）。
- **非对称注意力（v1.2.4）**：**仅 mask 位置产生 Query**；**Key/Value** 来自 `concat(CHS, mask)`；可见性仍满足：mask 可读同组 CHS 与同组 mask，**CHS 不作为 Q**（与「Layer Q→Mask K 禁止」相容）。块间仍隔离。

## 主模型条件层索引 `target_layer_ids`

实现见 `build_flashmtp_target_layer_ids(num_target_transformer_layers, num_draft_layers)`（`specforge/modeling/draft/flashmtp.py`），在 HF `hidden_states` 下标上选取：

- **必含**：索引 `0`（embedding 输出）、索引 `n−1` 与 `n`（最后两层 decoder 输出，其中 `n = num_target_transformer_layers` 为 decoder 层数）。
- **可选中间层**：当 `num_draft_layers > 0` 且区间 `[1, n−2]` 非空时，在该区间内 **均匀取 `num_draft_layers` 个点**（含端点插值）；与上合并后 **排序去重**。
- 若在 `flashmtp_config` 中显式提供 `target_layer_ids`，则直接使用该列表。

选中的层数记为 **L**（代码里 `num_target_layers` / `n_chs`），**不一定等于**主模型总层数。

## 张量与序列构造

- **Layer tokens**（总长 `N·L`，`N` 为 anchor 数）：**`tilde_z = RMSNorm(W_proj·H + E_layer[l])`**（`fc` 投影后与 `nn.Embedding` 逐槽相加，再 **`hidden_norm` 一次**）。与 mask 拼接后送入多层草稿 Transformer；**不再**在 decoder 内对 layer 段做门控加 `E_layer`（v1.2.2）。
- **Mask（noise） tokens**（总长 `N·B`，`B = block_size`）：`embed_tokens(mask 序列)`，**无全文绝对位置**；注意力中仅对 mask 的 Q/K 施加 **局部 RoPE**。
- **拼接时机（v1.2.4）**：`ctx_init` 与 mask **在每层 attention 前**拼成 `KV`；**Q 仅来自 `LN(mask)`**。最终 **`norm` 只作用于 mask 隐状态** 后接 `lm_head`。

## Layer token：CHS 布局与训练 / 推理数据源

### 布局（per-anchor 交错）

同一 anchor 上各层条件在序列中相邻：

`[layer0@a0, layer1@a0, …, layerL−1@a0, layer0@a1, …]`

实现：`torch.stack(selected_states, dim=2).reshape(B, N*L, H)`（`extract_context_feature` / `prepare_target_hidden` 一致）。

### 训练：`prepare_target_hidden`（`specforge/core/flashmtp.py`）

对每个 anchor，在序列维取 **`context_positions = (anchor_positions - 1).clamp(min=0)`** 处的各层 hidden，再按上式交错。即监督对齐在 **anchor 对应「前一 token」位置** 上的多深度表征。

### 推理：`extract_context_feature`（`specforge/modeling/draft/flashmtp.py`）

对当前 `forward` 得到的整段 `hidden_states` 做 **全序列** stack+reshape，得到 `(B, S·L, H)`，`S` 为当前主模型序列长度。`spec_generate` 中在 decode 步会对 `target_hidden` **按步切片**以匹配接受长度；若修改此处，须与 **交错布局（每步占 `L` 个连续向量）** 对齐，避免截断维数与训练语义不一致。

> **与 v1.1 的差异**：v1.1 曾用 `torch.cat(selected_states, dim=1)`（按层堆叠），与按 block 分组的 mask 不一致；v1.2 已改为交错。

### 为何需要 `E_layer`

- **消歧**：不同层的 `W_proj(H_l)` 统计域可能不同；可学习的 `E_layer[l]` 显式标记「第几层条件」（v1.2.2 起与投影结果相加后 **一次** `RMSNorm`）。
- **与 mask 分工**：layer 侧承载主模型多深度摘要；mask 侧用 **token id + 局部 RoPE** 表达块内草稿槽关系。

## 草稿网络结构（与「单层一次性」表述的更正）

草稿主体为 **`config.num_hidden_layers` 个** `Qwen3FlashMTPDecoderLayer` 顺序堆叠。每层：**`ctx_init` 固定**；`LN(mask)` 仅作 **Q**；`KV = concat(ctx_init, LN(mask))`；**post_attention_layernorm 与 MLP 仅作用于 mask**。**`attention_mask` 须由调用方预先构造**（训练：`OnlineFlashMTPModel`；推理：`spec_generate` 内调用 `build_flashmtp_dense_attn_bias`）。

## Attention 非对称与训练用稠密 mask（v1.2.4）

在 **仅 mask 为 Q** 时，允许的边体现在 **稠密加性 bias**（形状 `(B, 1, QM, KV_LEN)`）上，语义与旧版 `flashmtp_mask_mod` 一致：同组内 mask 可查 CHS 与 mask；块间不可见；无效块全屏蔽。**`build_flashmtp_dense_attn_bias`** 与 **`create_flashmtp_block_mask`**（FlexAttention 方形 QKV，旧全长自注意路径）同在 **`specforge/core/flashmtp.py`**；当前草稿注意力使用前者。

## Mask：局部 RoPE

- 每个 block 内 mask 的 `position_ids` 为 **`0, 1, …, B−1`**（`spec_generate` 用 `local_position_ids`；训练用 `_create_local_position_ids`）。
- **Q 全部为 mask**：对 **整段 Q** 施加 RoPE；**K** 拆成 CHS 段（无 RoPE）与 mask 段（RoPE），再拼接。
- **CHS** 仅作 **K/V 前缀**，深度信息来自 **`E_layer` 与 `W_proj·H` 融合后的 `ctx_init`**。


## Draft KV 与 Target KV

- **Target**：推理全程 `use_cache=True`，`DynamicCache`，步进与 `crop` 与现有自回归一致。
- **Draft**：`Qwen3FlashMTPAttention` 仍保留 `past_key_values` 接口以兼容 HF；**`spec_generate` 不传 draft cache**，每步对草稿做 **完整序列 forward**，与「草稿不跨步复用 KV」一致。

## 块内 logits 对齐（推理）

块内第 0 个位置为已知 anchor token，草稿预测 **第 `1…B−1` 个** 位置：`forward` 输出取 `[:, -B+1:, :]` 再经 `lm_head`（见 `spec_generate`）。

## 实现注意

- RoPE：**Q** 全为 mask（均旋转）；**K** 的 CHS 前缀不旋转、mask 段旋转。
- `chs_concat_mode` 已移除，无 feature/seq 开关。
- **`layer_ids`**：`torch.arange(L, device=...).repeat(n_blocks)`，与交错后的 `target_hidden` 长度 `n_blocks·L` 一致。
- **`flashmtp_config`** 常用键：`target_layer_ids`、`num_target_transformer_layers`、`mask_token_id` 等；训练脚本会写入与 `build_flashmtp_target_layer_ids` 一致的层列表。
