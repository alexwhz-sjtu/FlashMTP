# FlashMTP v3.3 推进文档

## 设计对齐（见 `FlashMTP_progress.md`）

- **阶段 1（MDLM）**：块内随机掩码率（可偏高掩码）、对掩码位置 CE；可选 KL（HF 目标模型 `logits`，`--kl-topk` 或全词表）；无距首位置 loss 衰减。
- **阶段 2（Streak）**：块首为 **clean token 嵌入**、块内其余为 `[MASK]`；streak 代理为 $\sum_{m}\exp(\sum_{j\le m}\log q_j)$，**块内仅 `pos_in_block>0`** 进入累加与外层求和（与 `FlashMTP_progress.md` v3.3 clarification）；最小化其负均值；`log_prob` 带下界 `clamp`；权重另含 `loss_mask` 与块有效区域。
- **数据**：目标模型生成响应，教师轨迹与 \(P\) 一致；CHS + Pivot 与现网 FlashMTP 一致。
- **衔接**：阶段 2 用 `--init-ckpt` 加载阶段 1 目录下的 `model.safetensors` 到**当前** `draft_model`，避免 checkpoint 内 `config.json` 中 `layer_types` 与 `num_hidden_layers` 不一致导致 `from_pretrained` 失败。


## 运行方式

```bash
./scripts/run_v3_3_training_flashmtp.sh --dt a800

./scripts/run_v3_3_mdlm.sh --dt a800
FLASHMTP_V33_INIT_CKPT=.../epoch_*_step_* ./scripts/run_v3_3_streak.sh --dt a800
./scripts/run_v3_3_pipeline.sh --dt a800
```

覆盖示例：`TRAIN_DATA_PATH`、`FLASHMTP_V33_TRAIN`、`TARGET_MODEL`、`CACHE_ROOT`、`OUTPUT_DIR_MDLM`、`NPROC_PER_NODE`、`WANDB_RUN_NAME` 等。  
**Epoch 分开控制**：`NUM_EPOCHS_MDLM`、`NUM_EPOCHS_STREAK` 分别传给对应阶段；未设时二者都回落到 `NUM_EPOCHS`（默认 6）。`STAMP` 中含 `_epm${NUM_EPOCHS_MDLM}_eps${NUM_EPOCHS_STREAK}` 以示区分。  
冒烟可 `export FLASHMTP_V33_TRAIN=./cache/smoke_v33/train.jsonl` 并保证文件存在。

## 验证

- 冒烟：`run_v3_3_pipeline.sh`（或单独 Streak + `FLASHMTP_V33_INIT_CKPT`）已在本地 GPU 上跑通；Streak 中 `log_probs` 与 `target_ids` 的 gather 已对齐为 `(B, N*bs)` 再 `view(B,N,bs)`。

## 总结

v3.3 已将 **MDLM 预训练** 与 **Streak 蒸馏** 分为两个独立 Python 入口与两套 shell，共享数据构建、存盘与 FSDP 逻辑；a800 下推荐使用 `run_v3_3_training_flashmtp.sh` 统一数据路径与双阶段输出目录。技术上通过 **对齐 layer_types** 与 **仅权重加载** 解决了阶段间衔接的 config 校验问题。
